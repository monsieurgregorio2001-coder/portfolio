Day 90 Deliverables (Hardening)

SLOs & Alerts

Service Level Objectives:

SLO Name Target
API Availability 99.9%
API Latency (P95) < 500ms
Background Job Success Rate 99.5%
Email Sync Latency < 5 min

Alerts Configured (PagerDuty or similar):

  • API error rate > 1% for 5 min
  • API latency P95 > 1s for 10 min
  • Database connections > 90%
  • AI costs > $500/hour
  • Background job failures > 5% for 30 min

Model Fallback & Traffic Splitting

Model Fallback:

  • Try OpenAI first
  • If rate_limit_exceeded, fallback to Anthropic
  • If both fail, throw error with helpful message

Traffic Splitting:

  • Route 90% to GPT-4o-mini (cheap)
  • Route 10% to GPT-4o (expensive, better quality)
  • Compare results
  • A/B test quality vs. cost

Circuit Breaker:

  • If error rate > 20%, stop calling AI
  • Fallback to non-AI features
  • Auto-recover when service restored

** Deliverable:** Resilient AI infrastructure

Definition of Done

Each deliverable must meet:

  • Code quality: Passes CI (lint, type-check, tests)
  • Testing: Unit tests + E2E tests for critical paths
  • Documentation: README updated, API docs current
  • Observability: Logging, metrics, tracing added
  • Security: Reviewed for common vulnerabilities
  • Performance: Meets latency requirements
  • Accessibility: WCAG 2.1 AA compliance (UI components)

This checklist ensures the team hit all requirements from the job description while building a production-ready system. Track progress weekly!
